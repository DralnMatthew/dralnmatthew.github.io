Finland Weather Prediction

  • Zhanghu Zhao

Abstract

Weather forecasting remains a complex and challenging task due to the myriad of variables involved and the vast amount of data generated. This project aims to address these challenges by implementing a sophisticated three-tiered architecture for real-time weather prediction. The system integrates data retrieval, secure storage, and rapid prediction components to enhance forecasting accuracy and efficiency. By utilizing live data from a weather API focused on Finland cities, the project seeks to build and refine machine learning models capable of generating accurate forecasts based on diverse and dynamic weather conditions. The initiative also explores advanced techniques in distributed computation and data visualization, with the goal of addressing critical questions regarding the optimization of real-time predictions and effective data handling in a rapidly changing environment.

Pipeline

In this data processing pipeline, Kafka is utilized to manage the flow of information between various components, centered around two main topics: weather and forecast. The OpenWeatherMap API feeds real-time weather data into the weather topic on Kafka, which acts as the initial data repository. Apache Spark Streaming then consumes this data from the weather topic to perform complex processing and analytics, including data cleaning, aggregation, and transformation. The processed results are published to the forecast topic. A Node.js server subscribes to the forecast topic to retrieve the updated information and leverages WebSocket technology to push the data to the frontend in real-time, enabling users to receive live updates directly in their web browsers. This architecture ensures a seamless end-to-end flow from data ingestion and processing to real-time delivery, leveraging Kafka for scalable data streaming, Spark for robust data processing, and WebSocket for efficient real-time communication with end-users.


Data

The operation involves refining weather data classification through a preprocessing step that consolidates several initial categories into a simplified set of final classes. Initially, weather conditions are categorized into Clear, Clouds, Mist, Fog, Haze, Smoke, Drizzle, Rain, Thunderstorm, and Snow. During preprocessing, Smoke, Haze, and Mist are grouped together under the Fog category to simplify classification. Additionally, Drizzle and Thunderstorm are merged into the Rain category, assuming that the distinctions between light precipitation and severe storms are less critical for the final classification. Consequently, the data is ultimately classified into five final classes: Clear, Clouds, Fog, Rain, and Snow, thereby streamlining the classification process and focusing on the most pertinent weather conditions.


Visualization

We use Node.js and d3.js to visualize the Finland map and weather data. A Node.js server subscribes to the forecast topic to retrieve updated information and leverages WebSocket technology to push the data to the frontend in real time, enabling users to receive live updates directly in their web browsers. d3.js is primarily used to render maps of Finland and visualize weather data. It loads geographic data in GeoJSON format and then scales and centers the map within SVG containers. Using D3's geoMercator projection and geoPath, the code draws the outlines of Finland's municipalities. When new weather data is received via a WebSocket, D3 dynamically creates circles on the map to represent current and forecasted weather conditions at various locations. The color and positioning of these circles are determined by temperature and weather type, enabling a real-time visual representation of the weather across Finland.

Usage

2. Run docker-compose up --build in the Terminal.

3. Visit http://localhost:8081
